# Telegram-бот и web CRM для контроля строительных контрактов

Проект теперь состоит из двух частей, которые могут работать на одной общей базе данных:

- Telegram-бот для быстрых действий, уведомлений и Excel-выгрузки;
- web CRM для работы через браузер с любого компьютера.

## Что умеет

- хранить контракты с датой окончания;
- хранить этапы внутри контракта;
- хранить оплаты по контрактам;
- показывать список контрактов и этапов;
- выгружать сводку в Excel;
- показывать ближайшие дедлайны по команде;
- отправлять автоматические напоминания в Telegram;
- открывать CRM в браузере с авторизацией и правами доступа по разделам.

## Команды

- `/start` - регистрация чата и краткая справка;
- `/new_contract` - пошаговое создание контракта;
- `/new_stage` - пошаговое создание этапа;
- `/new_payment` - добавить оплату по контракту;
- `/export_excel` - выгрузить Excel-отчет;
- `/contracts` - список всех контрактов и этапов;
- `/upcoming` - дедлайны на ближайшие 30 дней;
- `/upcoming 60` - дедлайны на ближайшие 60 дней;
- `/delete_contract ID` - удалить контракт;
- `/delete_stage ID` - удалить этап;
- `/cancel` - отменить текущий ввод.

## Установка

1. Установите Python 3.9+.
2. Создайте Telegram-бота через [@BotFather](https://t.me/BotFather) и получите токен.
3. В папке проекта создайте виртуальное окружение и установите зависимости:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

4. Создайте `.env` на основе `.env.example`:

```env
BOT_TOKEN=ваш_токен
BOT_TIMEZONE=Asia/Vladivostok
DB_PATH=contracts.db
```

5. Для локального запуска бота:

```bash
python3 bot.py
```

6. Для локального запуска web CRM:

```bash
python3 webapp.py
```

## Как использовать

1. Откройте чат с ботом и отправьте `/start`.
2. Добавьте контракт через `/new_contract`.
3. Добавьте этапы через `/new_stage`.
4. Проверяйте близкие сроки через `/upcoming`.
5. Бот будет присылать напоминания ежедневно в 09:00 по часовому поясу из `BOT_TIMEZONE`, если до конца осталось 14 или 7 дней.

## Локальная CRM

Чтобы открыть browser-версию локально:

```bash
python3 webapp.py
```

После запуска откройте:

```text
http://127.0.0.1:8000
```

Web CRM:
- читает те же данные из `DB_PATH` или `contracts.db`;
- показывает общий дашборд;
- показывает карточку контракта с этапами и оплатами;
- содержит разделы `Контракты`, `Аукционы`, `Доступы` и каркас будущих модулей;
- поддерживает авторизацию пользователей и права доступа по разделам.

## Railway

Если проект деплоится в Railway, один и тот же сервис теперь поднимает:

- Telegram-бота;
- web CRM на публичном URL Railway.

Для этого используется файл [`railway.json`](/Users/eduard25rus/.codex/worktrees/93a1/tgbot/railway.json), который запускает:

```bash
python3 serve.py
```

`serve.py`:
- поднимает web CRM на `0.0.0.0:$PORT`;
- параллельно запускает Telegram-бота;
- использует ту же `DB_PATH`, что и бот.

Для безопасного деплоя на Railway:

1. Подключите `Volume`.
2. Оставьте или задайте:

```env
DB_PATH=/app/data/contracts.db
BOT_TOKEN=...
BOT_TIMEZONE=Asia/Vladivostok
```

Для ежедневного автоимпорта 1С-выписок из почты добавьте переменные:

```env
BANK_MAIL_OWNER_CHAT_ID=329457823
BANK_MAIL_IMAP_HOST=imap.mail.ru
BANK_MAIL_IMAP_PORT=993
BANK_MAIL_LOGIN=felis.bank.statement@mail.ru
BANK_MAIL_PASSWORD=пароль_приложения_mail_ru
BANK_MAIL_FOLDER=INBOX
BANK_MAIL_SEARCH=ALL
BANK_MAIL_SENDER=sberbusiness@sberbank.ru
BANK_MAIL_SUBJECT=Выписка по сч
BANK_MAIL_AUTO_IMPORT_ENABLED=1
BANK_MAIL_DAILY_TIME=08:30
BANK_MAIL_INTERVAL_MINUTES=30
BANK_MAIL_MAX_AGE_HOURS=36
BANK_MAIL_RUN_ON_STARTUP=1
```

При старте сервиса CRM сразу проверит почту, затем по умолчанию будет проверять
ее каждые 30 минут через `BANK_MAIL_INTERVAL_MINUTES`, чтобы успевать скачать
ссылки Сбера до истечения срока. Если поставить `BANK_MAIL_INTERVAL_MINUTES=0`,
CRM вернется к одному ежедневному запуску в `BANK_MAIL_DAILY_TIME` по часовому
поясу `BOT_TIMEZONE`. Старые письма по умолчанию не проверяются: `BANK_MAIL_MAX_AGE_HOURS=36`
отсекает протухшие ссылки из архивных писем, а значение `0` отключает это ограничение.
По умолчанию импорт смотрит только письма СберБизнеса с темой выписки, чтобы не
цеплять служебные и рекламные письма в этом же ящике.

3. Если хотите красивую правильную ссылку установки пароля для сотрудников, добавьте:

```env
PUBLIC_BASE_URL=https://ваш-публичный-url-railway
```

Если `PUBLIC_BASE_URL` не задан, CRM сама попробует собрать внешний адрес из запроса Railway.

## Безопасные правки и быстрый откат

Перед изменениями в CRM полезно держать правки маленькими и проверяемыми:

```bash
git status
PYTHONPYCACHEPREFIX=/tmp/pycache python3 -m py_compile webapp.py storage.py bot.py serve.py scripts/smoke_webapp.py
python3 scripts/smoke_webapp.py
```

`scripts/smoke_webapp.py` создает временную SQLite-базу, тестового админа и открывает основные страницы CRM без запуска сервера. Если какая-то страница падает с 500, скрипт завершится с ошибкой до деплоя.

После успешной проверки лучше фиксировать изменения небольшими коммитами:

```bash
git add webapp.py storage.py README.md scripts/smoke_webapp.py
git commit -m "Короткое описание изменения"
git push origin main
```

Если после деплоя что-то сломалось, быстрые варианты отката:

- в Railway открыть предыдущий успешный deploy и нажать redeploy/rollback;
- либо сделать обратный коммит локально: `git revert <hash_проблемного_коммита>` и снова `git push origin main`.

## Хранение данных

- Все данные хранятся локально в SQLite-файле `contracts.db`.
- В Railway данные нужно хранить в SQLite-файле внутри `Volume`, иначе база может потеряться при новом деплое.
- Telegram-бот и web CRM читают одну и ту же базу, если у них одинаковый `DB_PATH`.
